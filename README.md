# warped-disk-xray

> **Data science toolkit for studying warped black hole accretion disks via X-ray polarization**

Based on: *Abarr & Krawczynski (2019) — "The Polarization of X-rays from Warped Black Hole Accretion Disks"* ([arXiv:1912.00243](https://arxiv.org/abs/1912.00243))

This repo addresses **6 research gaps** left open by the paper, using machine learning, Bayesian inference, and data pipelines designed to work with IXPE observational data (launched 2021).

---

## Research Gaps → Modules

| # | Gap from Paper | Module | Key Method |
|---|---------------|--------|-----------|
| 1 | Single fixed configuration studied | `src/parameter_sweep/` | Latin Hypercube Sampling |
| 2 | No inverse problem solver | `src/inference/` | MCMC / Bayesian posterior |
| 3 | Warped vs flat disk indistinguishable by eye (Fig. 9) | `src/classifier/` | 1D-CNN classifier |
| 4 | Double-reflection component unquantified | `src/decomposition/` | NMF + neural decomposer |
| 5 | Ray-tracing too slow for fitting | `src/emulator/` | Neural surrogate model |
| 6 | Simple geometry, not GRMHD-connected | `src/grmhd_bridge/` | HAMR output pipeline |

## Quickstart

### 1. Setup

```bash
git clone https://github.com/yourname/bp-xray-research.git
cd bp-xray-research

conda env create -f environment.yml
conda activate bp-xray

pip install -e .
```

### 2. Generate training data

```bash
# Runs parameter sweep with mock simulator
# Replace mock_simulate() with your ray-tracing wrapper
python src/parameter_sweep/generate_grid.py --config configs/sweep_config.yaml
```

### 3. Train the surrogate emulator

```bash
python src/emulator/train.py --config configs/emulator_config.yaml
```

### 4. Run Bayesian inference on an observed spectrum

```bash
python src/inference/run_mcmc.py \
    --spectrum data/processed/your_obs.npy \
    --emulator results/models/emulator_best.pt
# → outputs posterior JSON + corner plot
```

### 5. Classify warped vs flat disk

```bash
python src/classifier/train_classifier.py \
    --warped_data data/simulated/sweep_dataset.h5 \
    --flat_data   data/simulated/flat_dataset.h5 \
    --snr 20
```

### 6. Bridge GRMHD simulation to polarization

```bash
python src/grmhd_bridge/grmhd_to_polarization.py \
    --grmhd    data/raw/hamr_output.h5 \
    --emulator results/models/emulator_best.pt \
    --reader   hamr
```

### 7. Run tests

```bash
pytest tests/ -v
```

## Data Formats

All data is generated locally — nothing is committed to git.

**Spectra** are stored as `.npy` arrays of shape `(N_energy, 3)`:

| Column | Quantity | Unit |
|--------|----------|------|
| 0 | Energy | keV |
| 1 | Polarization fraction | % |
| 2 | Polarization angle | degrees |

Each `.npy` file has a companion `.json` with physical parameters:
```json
{
  "spin": 0.9,
  "r_bp": 8.0,
  "beta": 15.0,
  "phi": 90.0,
  "inclination": 75.0
}
```

**Datasets** (collections of spectra) are stored as HDF5:
```
dataset.h5
  ├── spectra   (N, N_energy, 3)
  └── params    (N, 5)
```

## Physical Background

The **Bardeen-Petters (BP)  ** arises when the black hole spin and accretion disk angular momenta are misaligned. General relativistic frame-dragging (Lense-Thirring precession) warps or breaks the disk at the **break radius r_BP**.

Key parameters encoded in the X-ray polarization signal:

| Parameter | Symbol | Range | Physical meaning |
|-----------|--------|-------|-----------------|
| BH spin | *a* | 0 – 0.998 | Frame-dragging strength |
| Break radius | r_BP | 3 – 50 r_g | Inner/outer disk boundary |
| Misalignment | β | 0° – 45° | Tilt between disk planes |
| Azimuthal angle | φ | 0° – 360° | Observer orientation |
| Inclination | i | 10° – 85° | Line-of-sight angle |

The paper's key finding: **polarization angle** (not fraction) carries the clearest signature of disk warping, and can provide a lower limit on β when the jet direction is known.

## Connecting to IXPE Data

IXPE (launched December 2021) measures 2–8 keV polarization. Public data archive:
[https://heasarc.gsfc.nasa.gov/docs/ixpe/archive/](https://heasarc.gsfc.nasa.gov/docs/ixpe/archive/)

Candidate targets from the paper:
- **GRO J1655-40** — jet at 85°, binary at 70°, misalignment ≥15°
- **Cygnus X-1** — inner disk vs binary plane disagree by ~10–15°
- **4U 1957+11** — consistently soft state, high spin candidate

See `docs/ixpe_data_guide.md` for the processing pipeline.

## Replacing the Mock Simulator

The file `src/parameter_sweep/generate_grid.py` contains `mock_simulate()` as a placeholder. To use your actual ray-tracing code:

```python
# In generate_grid.py, replace:
def mock_simulate(spin, r_bp, beta, phi, inclination) -> np.ndarray:
    ...

# With a wrapper like:
import subprocess, tempfile, numpy as np

def mock_simulate(spin, r_bp, beta, phi, inclination) -> np.ndarray:
    with tempfile.NamedTemporaryFile(suffix='.npy') as f:
        subprocess.run([
            './raytrace_binary',
            f'--spin={spin}', f'--r_bp={r_bp}',
            f'--beta={beta}', f'--phi={phi}',
            f'--incl={inclination}',
            f'--output={f.name}'
        ], check=True)
        return np.load(f.name)
```

See `docs/adding_a_simulator.md` for full instructions.
